# Editors

Editors are a very important tool in the world of programmers. Some very popular choices include but are not limited to

- [Visual studio code](Visual%20studio%20code.md) 

- [Vim](Vim.md) 

- [Neovim](NeoVim.md) 

- Intellij / pycharm / webstorm

- Android studio

- notepad++

I have personally used **notepad++** when starting out and later on switched to **Visual studio code**. Although I have worked with jetbrains IDEs (intellij, pycharm, webstorm, etc.). Currently I prefer using visual studio code but I am planning to start learning Vim to switch over to neovim after watching the insane productivity of some of it’s users.

To view my settings you can go to [my profile](https://github.com/thebetar/thebetar) in github.

### [Visual studio code](Visual%20studio%20code.md)

Visual studio code is a widely used IDE and is the best editor to start with when you start programming. It has a lot of features while also be extensible with some great choices. Visual studio code also contains a very easy to use debugger which ties into it’s terminal. The downside of Visual studio code is that it is written in Javscript using electron. This causes it to not perform on the levels of neovim for instance. Also the shortcuts are more limited than other choices.

### [Vim](Vim.md)

Vim is an IDE known for it’s steep learning curve and enormous amount of key bindings. This makes it very hard to pick up. But while researching the advantages a lot of fans of the IDE start to appear that have put in the effort to learn the key bindings and have achieved great productivity.

### [Neovim](NeoVim.md)

Neovim is a fork taken from Vim by developers that wanted to make a more extensible version of Vim. They kept the ideas about motions and minimal design from the get go but have created a version of Vim written in Lua and not vimscript which makes it easier to write extensions. This is also why this is a more preferred way than vim since it brings all the advantages of vim with the addition of being super extensible.
