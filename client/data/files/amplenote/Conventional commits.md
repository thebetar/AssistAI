# Conventional commits

Conventional commits require your commit messages to be structured with one of a couple options of **subjects** which describe the size of the change. The most used are `feat` and `fix` which describe an added feature or a bugfix. These subjects can also be appended by a `!` which means that the commit contains a breaking change. A short list of all the subjects are

- `feat:` a new feature has been added

- `fix:` a bug has been patched

- `build:` a change to the build of the project

- `ci:` a change to the pipeline or something relating

- `docs:` a change to the documentation

- `perf:` a performance improvement

- `refactor:` a code refactor

- `style:` a change to styling

- `test:` a change to the tests

- `chore:` everything that does not fall under the above

Source: [https://www.conventionalcommits.org](https://www.conventionalcommits.org/en/v1.0.0/)
