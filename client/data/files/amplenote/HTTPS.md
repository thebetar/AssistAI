# HTTPS

Https is used to secure the data being sent from the client to the server and from the server to the client. For this a public key, private key and certificate are used. But what are these and how will these be used?

The public key is used to encrypt the data from the clients side and sent it to the server which will decrypt the data using its private key.

The private key is used to decrypt data sent by the client and encrypt data sent from the server to the client.

The certificate is used to proof that the server is trustworthy this is generated by a trusted organisation. This is done by generating a **self signing certificate** and sending it to the organistation. The certificate contains data from the organisation that should be on the certificate, once received by the provider it will generate a certificate for the server which can be used to secure SSL.

\

To use HTTPS a certificate is needed with the extension `.crt` and a private key with the extension `.key`. The self signing certificate used for generating this certificate has the extension `.csr`
