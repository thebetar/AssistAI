# Frameworks

There are many frameworks within the frontend world, there are even jokes about the ever increasing amount of very useful and very useless frameworks. I have tried approximately 10 frameworks. My favourites out of all these experiences are `React` , `Vue` and `Svelte` .

For every framework I will go more in depth on what libraries I like to use in combination with these frameworks. But here are some short summaries of what I like about each.

## [Vue](Vue.md)

The strong point about Vue is how it uses its own `.vue` files. Within these files they have separated the templating, the scripting and the styling. I really like this because it keeps all the logic of a component within one file. This makes it really easy to work on styling and logic at the same time. Furthermore I have experienced the fastest idea to actual product with `Vue`

## [React](React.md)

React is more of a logical framework. Everything is done within `.jsx` or `.tsx` files. This makes it my preferable framework when working on applications where the visual aspect is less important and where the logic is more complex. Since everything is done within a function or class.

## [Astro](Astro.md)

Astro is a framework that is used for creating insanely fast static websites. This framework is the best choice when reactivity and scalablity aren’t a concern. This makes it perfect for portfolio website, like my own website [VonkProgramming.nl](https://vonkprogramming.nl) 

## Svelte

I have not used Svelte enough to really have a strong opinion about it yet. From what I saw it is really useful for application that do not have to scale into something big (e.g. small projects)

**Honorable mentions:**

- **Angular**: The big advantage about angular is that it includes all the necessary functionality like sending HTTP requests and handling data within it. This makes it perfect when working with large teams. The downside of Angular is that it is made in a way where scale should always be kept in mind, this makes the time from the drawing table to an actual application the longest of any framework, even though in my opinion if you make your `Vue` or `React` application in the right way they scale up just as well. Since I am not working in large teams it is not preferred by me.

- **Lit**: This works with web components which are now native to the browser. This makes it really performant. Also every component handles its own state and is completely independent. This is a strength and a weakness, because on the one hand your component clearly defines what it does and nothing else but in my experience it makes sharing data between components really hard.
