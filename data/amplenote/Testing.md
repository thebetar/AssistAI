# Testing

Testing is an integral part of creating resilient software. To ensure that everything works and keeps working a developer has to write tests. I have to admit based on the size en time constraints of a project there are differing levels of testing that can be done but one thing is for sure, you should always at least implement one of these ways of testing.

### Ways of testing

- [Unit testing](Unit%20testing.md) 

- [Integration testing](Integration%20and%20e2e%20testing.md) 

- [e2e testing](Integration%20and%20e2e%20testing.md) 

And if you want to be even more thorough there is also

- Data faking (faker-js/faker)

- Mutation testing (stryker.js)
