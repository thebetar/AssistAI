# Content management system

Content management systems are a great easy way to create APIs for your data while also allowing easy access to the data via a dashboard. This is amazing since in a lot of instances the platform you are building will be managed by someone without the technical know how to use a CLI tool to interact with the database.

## [Strapi](Strapi.md)

The most well known headless CMS solution within the javascript space is strapi. It offers an easily configurable solution to creating a backend with authentication and a dashboard to access the data stored. Strapi is also very customizable. A downside I have experienced is that it does not support mongodb in the latest version which is a database that fits really well into a lot of stacks.

## Payload CMS

Payload CMS is a new content management system that was created to be very barebones and easily customisable. The standard database used is MongoDB. This makes it very useful for developers using the MERN, MEVN or MEAN stack. This content management system is very new though so a lot of learning and fixing will have to be done by the developer.
