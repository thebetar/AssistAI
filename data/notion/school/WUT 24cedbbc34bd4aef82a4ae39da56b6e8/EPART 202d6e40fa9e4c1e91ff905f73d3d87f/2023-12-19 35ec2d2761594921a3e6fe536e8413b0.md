This lecture is about **decision trees**. With making decision trees it is important to create decisions that split the data well and evenly. It should also be taken into account that some checking could be destructive for the data and has to be saved to the end for instance when trying to classify a fruit, tasting it will damage the fruit.

A decision tree can be binary or multi-valued. This means that every layer a decision is made between two posibilities or multiple.

A tree should not become too large, this means that it contains too many nodes that have to make a decision so too large can refer to any dimension.
